回帰データを再作成・上書する機械

-環境------------------------------

マシン:HQA1B570
ユーザ:A1B5MEMO 
運用:リモートデスクトップ（パスワード含め、ナイスガイと同じ入り方です）

-ディレクトリ群:-----------------------

D:\
├reg_reflesh	プロダクト置いて、バッチを蹴るところです。
│├logs	各種実行ログが吐かれます。(クリアしないと累積です)
││├autotest_logs		初回、エビデンスを吐くための自動テストのログ置き場です。
││├rdcreate_logs		回帰データ作成のログ置き場です。
││├confirmtest_logs	確認のために再度行う、回帰テストのログ置き場です。
││└svndiff_logs	最後にとるSVNとの差分ログの置き場です。
│├tools	実行するための各種ツールが入っています。
││
│├※各種プロダクトの
│└※プロジェクトフォルダを配置してください。
│
└evidence	回帰データの元となる、エビデンスを出力するところです。(実行度に消されまス)
　│
　└※各種プロダクト名のフォルダ	自動生成されます
　　├input	元の回帰データ(TR_)を、コピーしリネームしたもの(TP_)が出力されます。
　　└output	新しい回帰データの元となるエビデンス(TR_)が出力されます。

-運用方法:--------------------------

・ほぼナイスガイのビルド環境と同様です。
　reg_reflesh に、プロダクトをガスガス放り込み、お好みで"x_SVN更新.bat"を蹴った後、
　"00.開始.bat"で実行してください。

・実行が完了しましたら、人間系でログの確認を行ってください。
　再回帰テストでこけている、カバレッジが極端に低い、などを確認、かと思います。

・確認が終わると、SVNにコミットかと思います。
　SVN差分のログを、何かウマイコト編集したら、一括コミットバッチが作れるかも？

-既知の問題-------------------------

・ツールのシェイプが足りない(jarを二百個ほど使う仕様)
・今以外の環境で動かない。(ドライブなど、決め打ちが多い)
・coberturaのXMLを吐かない(設定しているのに)
・フェイズごとに全プロダクトを回す形のため、最後まで待たなくては確認しにくい。

-他マシンへのインストール方法-------

□前提

以下の環境が整ったマシンが必要です。

・java1.5
・Subversion
・Maven2
・SPRITSの標準開発環境(Eclispeは必要ない)

□手順

①「環境(Dドライブ直下を想定）」のフォルダの内容物を、
対象マシンのD:\直下においてください。

②「ツールのソース」フォルダから、maven-rdconv-plugin を任意のフォルダに置き、
以下のコマンドを叩いて下さい。

 mvn clean compile install

③ローカルリポジトリ(普通は [userフォルダ]\.m2\repository)にある、
\murata\murata-app-parent の最新バージョンのpom(murata-app-parent-?.?.??.pom)に、
以下の文言を付け加えてください。

※<plugins></plugins>タグ内

		     <plugin>
		       <groupId>murata</groupId>
		       <artifactId>maven-rdconv-plugin</artifactId>
		       <version>1.0.1</version>
		     </plugin>
